## 1. 背景課題
複数のプレス機械を用いた金属部品加工の高サイクル生産を行っているS社では、
摩耗や潤滑不良など微細な設備異常がコスト損失を生じる課題を抱えている。
加工品質の安定化と設備保全の高度化を実現するため設備故障要因である異常
(特に潤滑不足)を早期に検知することを目指している。<br>

### 3.5 スコア改善
実施した前処理は以下の通りである。<br>
- **(a1)'-'除去**
- **(a2)平均・分散補正**：正常時の平均・分散値による補正
- **(a3)部品種別補正**：部品種別差のzスコア補正
- **(a4)異常値除去**：IQR(四分位)範囲外の値補正
- **(a5)ノイズ低減**：移動平均windows=5

ノイズ除去、zスコア補正, IQR補正, 移動平均の前処理を行うことにより、
前処理実行前からF1スコアが0.4256から0.8951へ向上。<br>

また実施した特徴量加工は以下の通りである。<br>
- **(b1)時系列変化取得**：<br>
スライド窓windows=30(前処理値そのまま)
- **(b2)時系列変化取得**：<br>
スライド窓windows=30から統計量(平均、分散、最大、最小、前データからの平均値変化量等)
- **(b3)物理的意味の強化データ**：<br>
速度補正振動(vibration_rms_g / spm)、BDC集中度(ftr_bdc_ratio / ftr_halfwidth80_deg)等

スライド窓や特徴量追加により、実行前からF1スコアが0.8951から
0.9354へ向上。スライド窓(SW=30)の値そのものを使用する場合と、
スライド窓(SW=30)データから統計量を作成する場合とでLightGBMモデルでは
スコアに大きな差は見られなかった。<br>

### 3.6 結果の考察
20日間の2件の異常を早期に検知できた。設備の故障はこれから定義が必要だが、
故障前のセンサーデータ異常を安定して捉えた。<br>
- **課題**：<br>
センサーデータ異常発生からの設備故障判定までの期間を明確にする(製品加工品質、設備停止条件)。
- **成功基準**：
|故障前に兆候検知(異常発生後に早期検知)|ほぼ達成|
|過剰検知の抑制(偽陽性FPゼロ)|達成|

## 4. 全体を通して
S社からのヒアリング情報に基づいたデータ選定とノイズ除去や製品種別差補正の
前処理が重要であると感じた。また時系列データ学習にスライド窓処理が効果的
である点を理解することができた。これは時系列データを行列で表現し画像的
(視覚的)なデータとして扱うことを意味する。<br>
今回教師有り学習モデルのLightGBMの二値分類性能が非常に良かった。しかし、
一般的に異常データは極端に少ない場合が多く、異常値の数を減らした場合は
KNN等の教師無し学習モデルでの二値分類がより効果を発揮するらしいので、
その解析も今度時間が有れば実施したい。<br>
